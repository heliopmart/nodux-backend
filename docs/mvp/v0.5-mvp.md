# Nodux v0.5 - Mínimo Produto Viável (Lean MVP)

**Legenda:**
* **Tutores:** Donos de animais de estimação.
* **Shadow Account:** Conta temporária criada pelo lojista para o tutor.
* **Timeline Bronze:** Histórico de saúde preenchido manualmente, sem validação externa.

---

## 1. Visão Geral e Objetivo
Validar a aceitação do pet shop em utilizar um dashboard digital em substituição à agenda de papel/WhatsApp e testar a taxa de conversão de tutores através do link de acompanhamento de status.

## 2. Escopo Reduzido (O que entra na v0.5)

### Módulo B2B (Pet Shop)   
* **Gestão de Identidade Única**: Sistema de login próprio (JWT) com troca de contexto (Dono vs. Tutor).
* **Multi-tenancy Lógica**: Separação de dados por `tenant_id` no banco de dados.
* **Dashboard de Hoje**: Lista simples de pets agendados para o dia com botão de "Dar Entrada".
* **Check-in Manual**: Cadastro rápido (Nome, Celular, Pet) que dispara o gatilho da Shadow Account.
* **Controle de Status (3 Estágios)**: Atualização manual: *Entrada* ➔ *Em Atendimento* ➔ *Pronto*.
* **Financeiro Simplificado**: Integração com Asaas para gerar cobranças avulsas (Pix/Boleto) sem split automático.

### Módulo B2C (Tutor)
* **Link WhatsApp (Experiência Lite)**: URL única para visualizar o status atual do pet e a timeline básica.
* **Conversão de Cadastro**: Botão para "Reivindicar Conta" e definir senha.
* **Timeline Bronze**: Visualização de vacinas e banhos cadastrados pelo pet shop.

---

## 3. O que foi REMOVIDO (Fica para a v1.0)
* **Selo Ouro (CRMV)**: Verificação de documentos e selos de autoridade clínica.
* **OCR de Vacinas**: Leitura automática de fotos de carteiras de vacina.
* **Status Preditivo**: Barra de progresso automática baseada em tempos médios.
* **Split de Pagamento**: Divisão automática de valores entre plataforma e lojista.
* **Agendamento Self-Service**: O tutor ainda não agenda sozinho; o lojista insere na agenda.

---

## 4. Stack Técnica e Infraestrutura v0.5

* **Backend**: Java 21 + Spring Boot 3.x (Spring Security + JWT).
* **Frontend**: Flutter (Web para o Pet Shop, Android/iOS para o Tutor).
* **Arquitetura de Banco**: 1 Instância EC2/VM rodando Docker com **PostgreSQL** e **Redis**.
* **Hospedagem**: AWS ou Azure (Instância básica de 2GB RAM).
